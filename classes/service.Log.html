static service.Log
since 1.1.0

A service to log events and happenings in a game. Use this service to collect statistics about a game session.

By default, the service records all triggers to every abilities loaded. The format of a log entry is: <timestamp>: <category> - <value>. For abilities, category is the ability's name, and the value is the value of pb-id of the element the ability acts on.

For every Protoboard component, region, and grid, you can customize the prefix of the id by specifying the pb-id attribute. When the element is activated, Protoboard will append a number to the attribute to make sure that they are unique. For instance:

<pb-c-card pb-id="attack"></pb-c-card>

will have its pb-id set to attack-0. So when the card is flipped, this service will log 3/28/2015, 6:01:48 PM: flip - attack-0.

For entries not handled by Abilities, you can also add log entries manually. Use the log method. For instance:

Protoboard.setup(function(require) {
  require('pb.service.Log').log('player-count', 2);
});

will generate entry: 3/28/2015, 6:01:48 PM: player-count - 2.

You can also customize the string output using the addPrinter and the addPreprocessor method. When a log entry is added, it undergoes two phases:

  1. Preprocess: The entry goes through all preprocessors that have been registered to the service. These preprocessors are used to add custom properties to the entry. Note that properties cannot be deleted or modified.
  2. Printing: The entry then goes through all registered printers. The service will use the non null value returned by the first printer.

For example:

Protoboard.setup(function(require) {
  var Log = require('pb.service.Log');
  Log
      .addPreprocessor(function(entry) {
        if (entry.category === 'player-count') {
          entry.set('count', entry.value);
        }
      })
      .addPrinter(function(entry) {
        if (entry.category === 'player-count') {
          return entry.timestamp.toLocaleString()
              + ': ' + entry.category + ' on ' + entry.count;
        } else {
          return null;
        }
      })
     .addPrinter(Log.defaultPrinter)
     .log('player-count', 2)
     .log('rng-seed', 1234);
});

A preprocessor and printer are functions that takes a log Entry. While a preprocessor only adds values to the Entry, a printer is expected to return a string, or a null.

You can specify more than one printer and preprocessors. These functions will be called in the order that they are added to the service. As soon as a printer returns a non null value, the values returned by the other printers in the chain will be ignored.

So the example above will generate:

3/28/2015, 6:21:47 PM: player-count on 2
3/28/2015, 6:21:53 PM: rng-seed - 1234

The custom preprocessor added the new count property to the entry, which was used by the custom printer to generate the entry for player-count. The default printer was used to generate the one for rng-seed.

You can retrieve the entries using blob as a blob, or entries as a string.
